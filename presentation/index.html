SecurityNode

Smart security that visually verifies threats before raising an alarm.

Jeronimo Vargas Orozco

jevargaso@unal.edu.co

What is SecurityNode?

An embedded security platform that detects physical intrusions and verifies threats using computer vision — running entirely on the edge, with no cloud dependency.

🚨 The Problem

Traditional alarms make noise, but they cannot tell you what triggered them. Pets, wind, or authorized users cause the same siren as an actual intruder.

👤 Who Uses It

Homeowners, small businesses, or anyone who wants an affordable, privacy-first security system that works without subscriptions or cloud storage.

⚙️ Main Function

Door contacts and PIR motion sensors detect intrusions. When triggered, the system captures an image and decides: real threat or false alarm?

📋 Expected Behavior

Alarm ON: Red LED + buzzer activate for suspicious conditions (covered face or timeout).

Alarm OFF: System logs the event and returns to idle — no alarm sounds.

No camera: Still works as a basic alarm (sensor → red LED + buzzer).

Top: ESP32-C3 (U3)  |  Center: ESP32-CAM Header (J3) with OV2640 camera  |  Bottom-left: Buzzer (LS1) + Red LED  |  Left: USB + Power Chain

Physical layout: USB connector at the board edge for easy access; ESP32-CAM header centered to allow ribbon cable clearance; buzzer and red LED positioned for visibility and audibility.

Enclosure idea: Small ABS case with USB cutout, lens hole for the camera module, LED window, and buzzer grill. Mounting holes included for mechanical integration.

Closing & Reflection

🔧 Greatest Technical Difficulty

Designing the power architecture to safely feed two ESP32 modules from a single USB source while adding overcurrent, ESD, and transient protection. Balancing the current budget and selecting regulators (TPS2553 for the CAM, AP2112K LDO for the controller) required careful trade-offs to ensure neither rail would collapse under peak load.

⭐ Best Resolved Part

The power tree. Separating the ESP32-CAM onto its own current-limited 5V branch while keeping the controller and logic on a clean 3.3V LDO rail ensures stability even if the vision module fails or is disconnected. The modular power design matches the modular system philosophy.

🚀 Next Version Improvement

Integrate Wi-Fi for remote monitoring and alerts; add MQTT or a lightweight web dashboard. Implement a more sophisticated vision model directly on the ESP32-CAM instead of relying on a laptop prototype. Add OTA firmware update capability.

📦 What Is Still Needed

Send the PCB for fabrication and assembly; validate the full system with the real ESP32-CAM module and its OV2640 camera; tune the face-coverage algorithm for real-world lighting conditions; design and 3D print an enclosure.

🎓 What I Learned

Electronic design is not just about choosing components — it is about anticipating failure modes. The decision to make the system degrade gracefully (fallback to basic alarm without the camera) taught me the importance of designing for reliability, not just functionality. A security system that fails silently is worse than one that fails loudly.
